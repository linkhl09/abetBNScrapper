# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A Selenium-based scraper that automates downloading student assignment submissions and quiz evidence from D2L Brightspace (`https://bloqueneon.uniandes.edu.co/`) for ABET accreditation evidence. For each configured activity it identifies the best (`mejor`) and worst (`peor`) graded submissions plus 3 randomly-selected students tracked across all activities (`Seguimiento_1/2/3`), downloads their evidence, renames it, and organizes it into folders. It also produces an Excel report of selected students and any download errors.

Activities are file-upload assignments (default), Grades-column D2L quizzes (`"type": "quiz"`), or quizzes with no Grades column at all (`"type": "standalone_quiz"`, reached via D2L's separate "Cuestionarios" tool). For the two quiz types there's no file to download, so the tool opens the student's most recent quiz attempt and saves the rendered page as a PDF instead.

There are no automated tests, build step, or linter configured in this repo.

## Running

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python scraper.py
```

Requires Google Chrome installed locally (Selenium drives a real Chrome window; login to Brightspace is manual — the script pauses with `input()` prompts for the user to log in and to navigate to the course if auto-navigation fails).

The script is interactive: it prompts in the terminal for the output root folder, course name/code, and section numbers, then opens Chrome, waits for manual login, navigates to the course's Grades page, scrapes it, and downloads submissions.

### Configuration files (not committed, created by the user at repo root)

- `activities.json` — list of `{name, aliases, groupName?, type?}` describing which gradebook columns (or, for `standalone_quiz`, which D2L quiz) to target (see `activitiesExample.json` for shape). `type` is `"assignment"` (default, omit the field), `"quiz"` (Grades-column quiz), or `"standalone_quiz"` (quiz with no Grades column, reached via the Cuestionarios tool; `groupName` is not supported for this type — startup validation exits if combined). If missing, the script prompts for activities interactively at startup — group-based and quiz-type activities can only be configured via this file, not interactively.
- `Groups_<course_code>.xlsx` — required only when any activity has a `groupName`. Must be exported from Brightspace's "Export Grades" with the "Group membership" and name checkboxes selected. Used to fall back to a group member's submission when the selected student didn't submit individually.
- `seguimiento.json` — optional. A flat JSON array of 1-3 student names (`"LastName, FirstName"`) to always use as the Seguimiento_1/2/3 students instead of the default random selection from the course roster. Validated by `D2LScraper._validate_seguimiento_override` at startup (fails fast on a bad shape).

## Architecture

Five modules, each with a single responsibility, wired together by `D2LScraper` in `scraper.py`:

- **`scraper.py`** — `D2LScraper` class owns the Selenium `webdriver` session and the whole run lifecycle (`run()`): login → per-section loop → scrape grades page HTML → parse → select students → download → standalone-quiz activities (once per section, see below). `download_student_evidence` dispatches per-activity on `activity_config["type"]`: `download_student_submissions` (assignments) redirects Chrome's native download behavior per-tab via `Page.setDownloadBehavior` CDP command pointed at the activity's destination folder, then `wait_and_rename_download` polls the folder for `.crdownload` completion and renames the result to `<type>-<Surnames>.zip`; `download_student_quiz_pdf` (Grades-column quizzes) instead waits for the quiz review SPA to render and calls the shared `_print_quiz_review_page_to_pdf` helper (wraps Selenium's `print_page()`, CDP `Page.printToPDF`) to save `<type>-<Surnames>.pdf` directly, no download/rename needed. Both paths share `_find_submission_link_with_fallback` (row lookup + group fallback) and `_open_link_in_new_tab`/`_close_tab_and_return`/`_recover_main_window` (tab bookkeeping). `download_standalone_quiz_evidence`/`download_standalone_quiz_pdf` (standalone quizzes) run a parallel, self-contained path: `_navigate_to_quizzes_tool` + `_find_quiz_mark_url` reach the quiz's `quiz_mark_users.d2l` grading page directly (no RPC token needed, `qi`/`ou` are plain query params on the quiz's list-page link), `parse_quiz_grading_page` (parser.py) extracts the student roster from that page directly (no Grades page involved at all), and `_find_last_quiz_attempt_link` locates each selected student's highest-numbered "intento N" link by D2L userId before reusing `_print_quiz_review_page_to_pdf`.
- **`parser.py`** — pure function `parse_grades_page(html, activities_config)` takes the raw grades page HTML (BeautifulSoup) and returns `{activity_name: [student_dict, ...]}`. It reconstructs multi-row `<th>` column headers (Brightspace grade columns often span two header rows with colspan/rowspan), matches column names against `activities_config` names/aliases, and extracts each student's name, id, numeric grade (`extract_numeric_grade`), and `col_idx` (position in the row — later used by `scraper.py` to re-locate the correct `<td>` when clicking through to a submission). `_find_grades_table`/`_extract_student_name_and_id` are shared helpers factored out of that function; `extract_course_roster(html)` reuses them to pull every student's `{name, id}` from the same Grades page independent of whether any activity column matched — used as the pool for picking the 3 Seguimiento students even when a section has no Grades-column activities at all. `parse_quiz_grading_page(html)` is the standalone-quiz counterpart: it walks a `quiz_mark_users.d2l` grading page's `<tr>`s directly (tracking the current student name from `d_gg` header rows), and returns one dict per student from that student's `D2LGridSummaryRow_List` summary row (`id` = D2L userId, `grade` via `extract_numeric_grade` again, `col_idx: None` — a deliberate no-op kept only so `select_students_for_activity` can be reused unmodified).
- **`utils.py`** — grab-bag of stateless, non-web helpers: interactive prompts (`prompt_user_config`, `load_activities`, `load_seguimiento_override`), student selection logic (`select_students_for_abet` picks 3 random unique students; `select_students_for_activity` picks best/worst by grade plus the abet-following students), name matching (`normalize_student_name` — order/comma-insensitive comparison key, needed because different D2L pages can render the same student's name in a different word order), and filename helpers (`sanitize_filename`, `extract_surnames` — expects `"LastName, FirstName"` format), `ensure_dir`.
- **`web_utils.py`** — generic Selenium/DOM helpers with no project-specific logic: `find_in_shadow` — a JS-injected recursive search through open shadow DOM trees, needed because Brightspace's UI is built with `d2l-*` custom elements (shadow DOM) that regular Selenium selectors can't pierce — plus `search_text_in_element_list` and `search_element_by_id`.
- **`excel_utils.py`** — thin `openpyxl` wrapper for building/appending to Excel tables (`create_workbook_with_sheet`, `add_sheet_with_table`, `append_array_to_table`) used for the results workbook (`results_<course_code>.xlsx`, saved to the root folder at the end of `run()` even on error via `finally`), plus `extract_student_groups` which parses a `Groups_<course_code>.xlsx` roster export into `{groupCategory: {groupName: [studentNames]}}`.

### Key data flow

`activities.json` config + scraped HTML → `parse_grades_page` → `{activity: [students]}`, in parallel `extract_course_roster` → full roster → `select_students_for_abet` (or, if `seguimiento.json` is present, `D2LScraper._build_seguimiento_from_override` instead — always wins) → `abet_students`, shared across every activity in the section → `select_students_for_activity` per matched activity → per-student `download_student_evidence` → `download_student_submissions` or `download_student_quiz_pdf` depending on activity `type` (locates row by name, cell by `col_idx` via `_find_submission_link_with_fallback`, opens the link in a new tab via `_open_link_in_new_tab`, then either downloads+renames a file or renders+prints a PDF) → results appended to the in-memory `openpyxl` workbook → workbook saved once at the end of the run.

Assignment cells expose an `onclick` containing `GetAssignmentLocation...`; quiz cells expose `QuizMarkNewTab(quizId, userId)` instead — `_get_submission_link_from_row` picks the right xpath marker based on `activity_type`. The quiz link's target URL already includes `selectedItem=mostRecent` (set by D2L itself), so the "last attempt" is loaded by default with no need to touch the attempt-switcher dropdown on that page.

### Standalone-quiz data flow

Runs once per section, after the Grades-column activity loop, for every activity with `type == "standalone_quiz"` (`run()`, right before the `finally:` block): `_navigate_to_quizzes_tool` → `_find_quiz_mark_url` (matches the quiz's name/aliases against the Cuestionarios list, then builds the `quiz_mark_users.d2l?qi=&ou=` URL straight from the regex-extracted query params on that quiz's own list-page link — no click-through needed) → `driver.get()` that URL → `parse_quiz_grading_page` → `select_students_for_activity(students, abet_students)` (same section-scoped `abet_students` the Grades-column activities used — see Key data flow above; it's populated from the full course roster or the `seguimiento.json` override, so it's never empty just because a section has zero Grades-column activities) → per selected student, re-resolve their D2L userId **by name** (via `normalize_student_name`, not an exact string match — see Fragile points) from the freshly-parsed list (never trust a cross-page `id`, mirrors how the Grades-column path always relocates students by name too) → `_find_last_quiz_attempt_link(user_id)` picks the highest-numbered "intento N" link (distinguished from the per-student "calificación general" summary link by the `actionParam='mark,` vs `actionParam='markoverall,` onclick substring) → click it (same-tab navigation, no new window opens) → `_print_quiz_review_page_to_pdf` → `driver.get(quiz_mark_url)` again to reset to a known page before the next student (deliberately not `driver.back()` — see Fragile points).

### Group fallback behavior

If a selected student has no submission/quiz link for a `groupName` activity, `_find_submission_link_with_fallback` looks up the student's group (from `self.groups_config`, populated by `extract_student_groups` in `D2LScraper._validate_groups_config`) and tries every other group member's row/column until it finds one with a link. If none of the group has submitted, it's logged to the `errors` sheet instead. This logic is type-agnostic (works the same for assignment and quiz links).

### Fragile points to be aware of when modifying

- Brightspace grades table lookup falls back between `#z_bk` and `table.d2l-grid` — both selector forms have been observed in the wild.
- `col_idx` from `parser.py` is reused later in `scraper.py` to index into `<td>`/`<th>` cells of a student's row — if the row structure differs from the header row (e.g. extra leading cells), indices will misalign.
- Download detection in `wait_and_rename_download` has two modes: match by cleaned/lowercased original filename, or (if filename unknown) grab the first file in the folder that isn't a `.crdownload` or an already-renamed target — the latter mode assumes only one download lands in that folder at a time.
- `_print_quiz_review_page_to_pdf`'s wait for the quiz review page (a `<d2l-consistent-evaluation>` shadow-DOM SPA, same component family the assignment path pierces via `find_in_shadow` for the download button) uses a structural `find_in_shadow` gate plus a fixed `time.sleep(6)` heuristic for content to finish rendering before printing (raised from an initial `3`s after live testing caught a PDF printed before the page had loaded) — if generated PDFs still come out blank/partial on slower quizzes, increase the sleep further or add a scroll-to-bottom-then-top nudge before calling `print_page()`. This same helper is reused by the standalone-quiz path, whose destination page was confirmed to be the same review SPA only by the user's own manual-workflow description, not by an independent HTML capture — re-check this first if standalone-quiz PDFs come out empty.
- `download_standalone_quiz_pdf` deliberately reloads `quiz_mark_users.d2l` via `driver.get()` after every student instead of `_close_tab_and_return`'s `driver.back()` fallback, because the "intento N" link is a `javascript://` + `D2L.NavInfo` trigger (not a normal href) and browser-history behavior for it was never verified live.
- `_find_quiz_mark_url` only searches the currently-rendered Cuestionarios list DOM; it does not follow pagination if the list has more than one page (unobserved in testing — the one course explored rendered 21 quizzes on a single unpaginated page with no pager markup at all). A quiz on a later page will currently just fail with a clear "not found" error.
- `abet_students` is deliberately sourced from `extract_course_roster` (the full Grades-page roster) rather than from `activity_data` (only students appearing in a matched activity column) — an earlier version used the latter, which meant a section with zero `type: "quiz"`/`"assignment"` entries (only `standalone_quiz`) got an empty Seguimiento selection, since `activity_data` is empty in that case. If this ever regresses, check that `run()` computes `course_roster`/`abet_students` unconditionally, before the `if not activity_data:` branch.
- `parse_quiz_grading_page`'s `d_gg` header row has TWO different shapes depending on whether the student has any attempts: a normal student's nested row is `<td>checkbox</td><td width="100%">Name</td>`, but a student with zero attempts (e.g. a manually-entered/overridden grade, D2L label "elementos sobreescritos manualmente") has no checkbox to select an attempt with, so the row is just `<td>Name</td>` — a single cell with no `width="100%"` attribute at all. The parser takes the LAST `<td>` in that nested row rather than filtering by `width="100%"`, specifically to handle this case — filtering by `width="100%"` silently dropped these students' names, which then caused their (real, sometimes lowest/highest) grade to be skipped entirely rather than just mis-attributed, corrupting mejor/peor selection. If this page's markup changes again, re-verify against a real capture rather than assuming the two known shapes are exhaustive.
- The Grades page and the Cuestionarios/Quizzing admin pages can render the same student's display name in a different word order (observed live: Grades page gave "FirstName LastName", the quiz page gave "LastName, FirstName" for the same student, for reasons not otherwise understood — possibly a per-view name-display setting). `download_standalone_quiz_evidence`'s by-name matching between `abet_students` (sourced from the Grades page) and the quiz's own parsed roster therefore goes through `normalize_student_name` (utils.py — tokenizes on whitespace/commas and sorts, order/comma-insensitive) instead of an exact string match; an exact match silently misreported every Seguimiento student as "did not attempt the quiz" when they actually had.
